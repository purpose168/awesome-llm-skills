# 文件组织器

此技能可作为您的个人组织助手，帮助您在计算机上维护整洁、合理的文件结构，无需持续手动组织的精神负担。

## 何时使用此技能

- 您的下载文件夹混乱不堪
- 由于文件散落各处，您找不到所需文件
- 您有占用空间的重复文件
- 您的文件夹结构不再合理
- 您想建立更好的组织习惯
- 您正在开始一个新项目，需要良好的结构
- 您在归档旧项目之前需要清理

## 此技能的功能

1. **分析当前结构**：检查您的文件夹和文件，了解您拥有的内容
2. **查找重复文件**：识别系统中的重复文件
3. **建议组织方案**：根据您的内容提出合理的文件夹结构
4. **自动化清理**：在您的批准下移动、重命名和组织文件
5. **保持上下文**：根据文件类型、日期和内容做出智能决策
6. **减少杂乱**：识别您可能不再需要的旧文件

## 使用说明

当用户请求文件组织帮助时：

1. **理解范围**
   
   询问明确的问题：
   - 哪个目录需要组织？（下载、文档、整个主文件夹？）
   - 主要问题是什么？（找不到东西、重复文件、太乱、没有结构？）
   - 有任何需要避免的文件或文件夹吗？（当前项目、敏感数据？）
   - 组织的积极程度如何？（保守型 vs. 全面清理）

2. **分析当前状态**
   
   检查目标目录：
   ```bash
   # 获取当前结构概览
   ls -la [target_directory]
   
   # 检查文件类型和大小
   find [target_directory] -type f -exec file {} \; | head -20
   
   # 识别最大的文件
   du -sh [target_directory]/* | sort -rh | head -20
   
   # 统计文件类型
   find [target_directory] -type f | sed 's/.*\.//' | sort | uniq -c | sort -rn
   ```
   
   总结发现：
   - 文件和文件夹总数
   - 文件类型分布
   - 大小分布
   - 日期范围
   - 明显的组织问题

3. **确定组织模式**
   
   根据文件，确定合理的分组方式：
   
   **按类型**：
   - 文档（PDF、DOCX、TXT）
   - 图片（JPG、PNG、SVG）
   - 视频（MP4、MOV）
   - 归档文件（ZIP、TAR、DMG）
   - 代码/项目（包含代码的目录）
   - 电子表格（XLSX、CSV）
   - 演示文稿（PPTX、KEY）
   
   **按用途**：
   - 工作 vs. 个人
   - 活跃 vs. 归档
   - 项目特定
   - 参考资料
   - 临时/草稿文件
   
   **按日期**：
   - 当前年份/月份
   - 前几年
   - 非常旧的（归档候选）

4. **查找重复文件**
   
   当用户请求时，搜索重复文件：
   ```bash
   # 通过哈希值查找完全重复的文件
   find [directory] -type f -exec md5 {} \; | sort | uniq -d
   
   # 查找名称相同的文件
   find [directory] -type f -printf '%f\n' | sort | uniq -d
   
   # 查找大小相似的文件
   find [directory] -type f -printf '%s %p\n' | sort -n
   ```
   
   对于每组重复文件：
   - 显示所有文件路径
   - 显示大小和修改日期
   - 建议保留哪个（通常是最新的或命名最好的）
   - **重要**：在删除前始终请求确认

5. **提出组织计划**
   
   在进行更改之前，提出清晰的计划：
   
   ```markdown
   # [目录]的组织计划
   
   ## 当前状态
   - X个文件分布在Y个文件夹中
   - 总计[大小]
   - 文件类型：[分布情况]
   - 问题：[列出问题]
   
   ## 建议的结构
   
   ```
   [目录]/
   ├── Work/（工作）
   │   ├── Projects/（项目）
   │   ├── Documents/（文档）
   │   └── Archive/（归档）
   ├── Personal/（个人）
   │   ├── Photos/（照片）
   │   ├── Documents/（文档）
   │   └── Media/（媒体）
   └── Downloads/（下载）
       ├── To-Sort/（待分类）
       └── Archive/（归档）
   ```
   
   ## 我将进行的更改
   
   1. **创建新文件夹**：[列表]
   2. **移动文件**：
      - X个PDF → Work/Documents/（工作/文档）
      - Y个图片 → Personal/Photos/（个人/照片）
      - Z个旧文件 → Archive/（归档）
   3. **重命名文件**：[任何重命名模式]
   4. **删除**：[重复文件或垃圾文件]
   
   ## 需要您决定的文件
   
   - [列出任何不确定的文件]
   
   准备开始吗？（是/否/修改）
   ```

6. **执行组织**
   
   获得批准后，系统地进行组织：
   
   ```bash
   # 创建文件夹结构
   mkdir -p "path/to/new/folders"
   
   # 移动文件并记录清晰日志
   mv "old/path/file.pdf" "new/path/file.pdf"
   
   # 使用一致的模式重命名文件
   # 示例："YYYY-MM-DD - 描述.ext"
   ```
   
   **重要规则**：
   - 在删除任何内容之前始终确认
   - 记录所有移动操作以便可能撤销
   - 保留原始修改日期
   - 优雅地处理文件名冲突
   - 如果遇到意外情况，停止并询问

7. **提供摘要和维护提示**
   
   组织完成后：
   
   ```markdown
   # 组织完成！✨
   
   ## 更改内容
   
   - 创建了[X]个新文件夹
   - 组织了[Y]个文件
   - 通过删除重复文件释放了[Z] GB空间
   - 归档了[W]个旧文件
   
   ## 新结构
   
   [显示新的文件夹树]
   
   ## 维护提示
   
   为了保持整洁：
   
   1. **每周**：对新下载的文件进行分类
   2. **每月**：审查并归档已完成的项目
   3. **每季度**：检查新的重复文件
   4. **每年**：归档旧文件
   
   ## 给您的快速命令
   
   ```bash
   # 查找本周修改的文件
   find . -type f -mtime -7
   
   # 按类型对下载进行分类
   [针对您设置的自定义命令]
   
   # 查找重复文件
   [自定义命令]
   ```
   
   想要组织另一个文件夹吗？
   ```

## 专业建议

1. **从小处着手**：从一个杂乱的文件夹（如下载文件夹）开始，建立信任
2. **定期维护**：每周对下载文件夹进行清理
3. **命名一致**：对重要文件使用"YYYY-MM-DD - 描述"格式
4. **积极归档**：将旧项目移动到Archive（归档）而不是删除
5. **分离活跃与归档**：保持活跃工作与归档工作之间的清晰界限
6. **信任流程**：让Claude处理"文件应该放在哪里"的认知负担

## 最佳实践

### 文件夹命名
- 使用清晰、描述性的名称
- 避免使用空格（使用连字符或下划线）
- 具体明确：使用"client-proposals"（客户提案）而不是"docs"（文档）
- 使用前缀排序："01-current"（01-当前）、"02-archive"（02-归档）

### 文件命名
- 包含日期："2024-10-17-meeting-notes.md"（2024-10-17-会议记录.md）
- 描述清晰："q3-financial-report.xlsx"（q3财务报告.xlsx）
- 避免在名称中使用版本号（使用版本控制代替）
- 移除下载痕迹："document-final-v2 (1).pdf"（文档最终版v2 (1).pdf）→ "document.pdf"（文档.pdf）

### 何时归档
- 6个月以上未触碰的项目
- 可能以后会参考的已完成工作
- 迁移到新系统后的旧版本
- 您犹豫是否删除的文件（先归档）
