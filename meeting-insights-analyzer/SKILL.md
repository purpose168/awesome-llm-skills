# 会议洞察分析器

此技能将您的会议记录转化为关于沟通模式的有价值洞察，帮助您成为更有效的沟通者和领导者。

## 何时使用此技能

- 分析跨多个会议的沟通模式
- 获得关于领导力和引导风格的反馈
- 识别何时回避困难对话
- 了解您的说话习惯和填充词使用
- 随时间跟踪沟通技能的改进
- 用具体示例准备绩效评估
- 指导团队成员的沟通风格

## 此技能的功能

1. **模式识别**：识别跨会议的重复行为，例如：
   - 冲突回避或间接沟通
   - 说话比例和轮流发言
   - 提问与陈述模式
   - 积极倾听指标
   - 决策方法

2. **沟通分析**：评估沟通效果：
   - 清晰度和直接性
   - 填充词和模糊语言的使用
   - 语气和情感模式
   - 会议控制和引导

3. **可操作反馈**：提供具体的、带时间戳的示例，包括：
   - 发生了什么
   - 为什么重要
   - 如何改进

4. **趋势跟踪**：在分析多个会议时比较随时间变化的模式

## 如何使用

### 基本设置

1. 将会议记录下载到文件夹（例如 `~/meetings/`）
2. 在 Claude Code 中导航到该文件夹
3. 请求您想要的分析

### 快速开始示例

```
分析此文件夹中的所有会议，告诉我何时回避了冲突。
```

```
查看我过去一个月的会议，识别我的沟通模式。
```

```
比较这两个会议文件夹之间的引导风格。
```

### 高级分析

```
分析此文件夹中的所有记录并：
1. 识别我何时打断他人
2. 计算我的说话比例
3. 找出我回避给出直接反馈的时刻
4. 跟踪我对填充词的使用
5. 展示良好积极倾听的示例
```

## 指令

当用户请求会议分析时：

1. **发现可用数据**
   - 扫描文件夹中的记录文件（.txt、.md、.vtt、.srt、.docx）
   - 检查文件是否包含说话者标签和时间戳
   - 确认会议的日期范围
   - 识别记录中用户的名称/标识符

2. **明确分析目标**

   如果未指定，询问他们想了解什么：
   - 特定行为（冲突回避、打断、填充词）
   - 沟通效果（清晰度、直接性、倾听）
   - 会议引导技能
   - 说话模式和比例
   - 需要改进的领域

3. **分析模式**

   对于每个请求的洞察：

   **冲突回避**：
   - 寻找模糊语言（"maybe"、"kind of"、"I think"）
   - 间接措辞而非直接请求
   - 出现紧张时转移话题
   - 无承诺地同意（"yeah, but..."）
   - 不解决明显的问题

   **说话比例**：
   - 计算会议中说话时间的百分比
   - 计算打断次数（用户打断他人和被他人打断）
   - 测量平均说话轮次长度
   - 跟踪提问与陈述的比例

   **填充词**：
   - 计算 "um"、"uh"、"like"、"you know"、"actually" 等词
   - 记录每分钟或每次说话轮次的频率
   - 识别它们增加的情况（紧张、不确定）

   **积极倾听**：
   - 引用他人之前观点的问题
   - 解释或总结他人想法
   - 建立在他人贡献基础上
   - 提出澄清问题

   **领导力和引导**：
   - 决策方法（指令式 vs. 协作式）
   - 如何处理分歧
   - 包含较少发言的参与者
   - 时间管理和议程控制
   - 后续跟进和行动项清晰度

4. **提供具体示例**

   对于发现的每个模式，包括：

   ```markdown
   ### [模式名称]

   **发现**：[模式的一句话总结]

   **频率**：[Y 个会议中出现 X 次]

   **示例**：

   1. **[会议名称/日期]** - [时间戳]

      **发生了什么**：
      > [记录中的实际引用]

      **为什么重要**：
      [对影响或错失机会的解释]

      **更好的方法**：
      [具体的替代措辞或行为]

   [重复 2-3 个最强的示例]
   ```

5. **综合洞察**

   分析完所有模式后，提供：

   ```markdown
   # 会议洞察总结

   **分析期间**：[日期范围]
   **已分析会议**：[X 个会议]
   **总时长**：[X 小时]

   ## 识别的关键模式

   ### 1. [主要模式]
   - **观察到的**：[您看到的]
   - **影响**：[为什么重要]
   - **建议**：[如何改进]

   ### 2. [次要模式]
   [相同结构]

   ## 沟通优势

   1. [带示例的优势 1]
   2. [带示例的优势 2]
   3. [带示例的优势 3]

   ## 成长机会

   1. **[领域 1]**：[具体的、可操作的建议]
   2. **[领域 2]**：[具体的、可操作的建议]
   3. **[领域 3]**：[具体的、可操作的建议]

   ## 说话统计

   - 平均说话时间：[会议的 X%]
   - 提出的问题：[平均每会议 X 个]
   - 填充词：[每分钟 X 个]
   - 打断：[每会议给出 X 次 / 收到 Y 次]

   ## 后续步骤

   [3-5 个改进沟通的具体行动]
   ```

6. **提供后续选项**
   - 在未来的会议中跟踪相同指标
   - 深入研究特定会议或模式
   - 与行业基准比较
   - 创建个人沟通发展计划
   - 为绩效评估生成总结

## 示例

### 示例 1：冲突回避分析（灵感来自 Dan Shipper）

**用户**："我下载了所有会议录音并将它们放在一个文件夹中。告诉我所有我微妙地回避冲突的次数。"

**输出**：
```markdown
# 冲突回避模式

在 15 个会议中发现 23 次您使用间接沟通或回避直接处理紧张关系的情况。

## 模式：对关键反馈的模糊处理

**频率**：7 个会议中出现 8 次

**示例 1：与 Sarah 的一对一会议** - 00:14:32

**发生了什么**：
> "所以，我在想……也许我们可以，比如，可能考虑再次查看时间表？我是说，如果你觉得这有道理。但无论你怎么想都是最好的！"

**为什么重要**：
您需要解决 Sarah 的项目落后于时间表的问题，但模糊语言（"maybe"、"potentially"、"if you think"）和最后的转移（"whatever you think"）使她很容易错过紧迫性。

**更好的方法**：
"Sarah，项目落后于时间表两周。我们需要讨论什么阻碍了进展，并在今天创建一个新的时间表。"

[其他示例...]

## 建议

1. **在第一句话中直接指出问题**
2. **删除模糊词**，如 "maybe"、"kind of"、"sort of"
3. **提出具体问题**而不是暗示
4. **安排困难对话**而不是随意提出
```

### 示例 2：领导力引导

**用户**："分析我的团队会议，告诉我关于我的引导风格。"

**输出**：提供关于以下内容的洞察：
- 您说话与团队成员说话的比例（60% vs. 40%）
- 您是提问还是陈述（3:1 比例）
- 您如何处理分歧（倾向于过快解决）
- 谁说得最少以及您是否让他们参与
- 良好和错失引导时刻的示例

### 示例 3：个人发展跟踪

**用户**："比较我第一季度与第二季度的会议，看看我是否改进了倾听技能。"

**输出**：创建比较分析，显示：
- 打断减少（每会议 8 次 → 3 次）
- 澄清问题增加（每会议 2 次 → 7 次）
- 建立在他人想法基础上的改进
- 显示差异的具体示例
- 仍需改进的领域

## 设置提示

### 获取会议记录

**来自 Granola**（订阅 Lenny 的时事通讯免费）：
- Granola 自动转录您的会议
- 将记录导出到文件夹：[操作说明]
- 将 Claude Code 指向该文件夹

**来自 Zoom**：
- 启用带转录的云录制
- 会议后下载 VTT 或 SRT 文件
- 存储在专用文件夹中

**来自 Google Meet**：
- 使用 Google Docs 自动转录
- 将记录文档保存到文件夹
- 下载为 .txt 文件或授予 Claude Code 访问权限

**来自 Fireflies.ai、Otter.ai 等**：
- 批量导出记录
- 存储在本地文件夹中
- 对文件夹运行分析

### 最佳实践

1. **一致的命名**：使用 `YYYY-MM-DD - 会议名称.txt` 格式
2. **定期分析**：每月或每季度审查趋势
3. **具体查询**：一次询问一个行为以获得深度
4. **隐私**：将敏感会议数据保留在本地
5. **以行动为导向**：一次专注于一个改进领域
