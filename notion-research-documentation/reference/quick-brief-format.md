# 快速简报格式

**何时使用**：
- 时间敏感的请求
- 简单主题
- 状态更新
- 快速参考需求

## 特征

**长度**：200-400 字

**结构**：
- 3-4 句话摘要
- 3-5 个关键要点
- 简短行动项列表
- 简短来源列表

## 模板

完整模板请参阅 [quick-brief-template.md](quick-brief-template.md)。

## 最适用于

- 快速周转请求
- 简单、直接的主题
- 快速状态更新
- 当时间比深度更重要时
- 深入研究之前的初步探索

## 示例用例

- "快速总结我们的 API 文档中的内容"
- "快速简报昨天的会议笔记"
- "那个规范的关键点是什么？"
- "给我一个项目状态的快速概述"
