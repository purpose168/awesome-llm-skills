# 格式选择指南

根据研究需求选择合适的输出格式。

## 决策树

```
这是在比较多个选项吗？
  ├─ 是 → 使用比较格式
  └─ 否 ↓

这是时间敏感或简单的吗？
  ├─ 是 → 使用快速简报
  └─ 否 ↓

这需要正式/广泛的文档吗？
  ├─ 是 → 使用综合报告
  └─ 否 → 使用研究摘要（默认）
```

## 格式指南

### 标题
- 使用 `#` 作为标题
- 使用 `##` 作为主要章节
- 使用 `###` 作为子章节
- 保持标题层次结构一致

### 列表
- 使用 `-` 作为项目符号
- 使用 `1.` 作为编号列表
- 保持列表项结构平行

### 强调
- 使用 `**粗体**` 表示关键术语和章节标签
- 使用 `*斜体*` 表示强调
- 适度使用以获得最大效果

### 引用
- 始终使用 `<mention-page url="...">页面标题</mention-page>` 引用来源页面
- 在引用的信息后立即包含引用
- 在末尾的"来源"章节中分组所有来源

### 表格
- 用于结构化数据比较
- 保持 3-5 列以提高可读性
- 包含标题行
- 适当对齐内容

### 代码块
在包含以下内容时使用：
- 技术规范
- 配置示例
- 命令示例

```
此处为示例代码或配置
```

## 内容指南

### 执行摘要
- 以最重要的发现开头
- 包含 1-2 个关键影响
- 使其独立（读者无需进一步阅读即可获得价值）
- 摘要目标为 2-3 句话，报告目标为 1 段

### 主要发现
- 以清晰的标题开头
- 用具体证据支持
- 包含相关数据点或引用
- 立即引用来源
- 专注于可操作的见解

### 建议
- 使其具体且可操作
- 解释每个建议背后的"原因"
- 明确确定优先级（优先级 1、2、3 或高/中/低）
- 在相关时包含实施提示

### 来源引用
- 使用提及链接到原始页面
- 注意信息是否过时（检查最后编辑日期）
- 引用时注明特定章节
- 将相关来源分组在一起
